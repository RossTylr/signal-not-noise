# 07 — Applied Thinking

Knowing the methods isn't enough. Knowing *when* to use them is the skill.

## Notebooks

| Notebook | Title | Core Question |
|----------|-------|---------------|
| **07a** | Decision Framework | Which method for which problem? |
| **07b** | Simulation Parameter Spaces | Dimensionality in DES and Monte Carlo |
| **07c** | Compression as Philosophy | Why Ng is right: think, don't brute force |

## The Arc

**07a** synthesises everything into a decision tree. Given a dataset and a goal, which dimensionality reduction method should you reach for? We build a flowchart and test it on multiple scenarios.

**07b** connects directly to discrete-event simulation. Your simulation's parameter space IS a high-dimensional space. Design of experiments, sensitivity analysis, and surrogate modelling are all dimensionality reduction in disguise. Monte Carlo variance reduction is compression.

**07c** zooms out to the philosophy. Ng's argument that compression beats brute force isn't just about ML — it's about engineering, science, and thinking itself. The best models aren't the biggest. They're the ones that found the right subspace.
