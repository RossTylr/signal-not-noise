# 01 — What Is a Dimension?

This module builds intuition from the ground up. No assumptions, no jargon.

## Notebooks

| Notebook | Title | Core Question |
|----------|-------|---------------|
| **01a** | Building Intuition | What actually *is* a dimension? |
| **01b** | The Curse of Dimensionality | Why does adding dimensions make things worse? |
| **01c** | Real-World High Dimensions | Where does high-dimensional data actually live? |

## The Arc

**01a** establishes that a dimension is just a question you can ask about your data — nothing mystical. We go from 1D to 10,000D and build comfort with the idea that maths doesn't care about our inability to visualise past 3D.

**01b** is where things get uncomfortable. We show — viscerally, with code and visuals — that high-dimensional spaces are pathological. Distances collapse, volumes vanish, and algorithms silently break. This is the motivation for everything that follows.

**01c** grounds it in reality: genomics, medical imaging, NLP embeddings, simulation parameter spaces. These aren't toy problems.

## After This Module You Should Be Able To

- Explain what "100-dimensional data" means to a non-technical person
- Describe three specific ways high dimensions break algorithms
- Articulate *why* dimensionality reduction is necessary, not just nice-to-have
